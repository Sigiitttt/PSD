Menangani Imbalance Ecoli dengan ADASYN#

Dokumentasi ini berisi proyek saya yang membahas salah satu topik penting di machine learning, yaitu ketidakseimbangan data (data imbalance). Studi kasus yang digunakan adalah dataset Ecoli dari UCI.

Tujuan utama dari proyek ini adalah untuk menyeimbangkan distribusi kelas pada dataset Ecoli menggunakan teknik oversampling ADASYN (Adaptive Synthetic Sampling), serta memvisualisasikan hasilnya.

Tahapan yang saya lakukan:#

  1. Menganalisis distribusi kelas data Ecoli.

  2. Memvisualisasikan data dengan PCA untuk melihat ketidakseimbangannya.

  3. Mengatasi batasan teknis dari algoritma ADASYN pada dataset ini.

  4. Menerapkan metode gabungan (RandomOverSampler + ADASYN) untuk menyeimbangkan data.

  5. Menganalisis hasil dari ADASYN dan menjelaskan mengapa jumlah sampelnya bervariasi.

  6. Melakukan plotting pada data yang sudah diimbangkan menggunakan PCA dan menandai data hasil generate sebagai bukti akhir.

Langkah 1 & 2: Analisis dan Persiapan Data#

Proses diawali dengan mengambil data Ecoli dari database MySQL lokal. Setelah data dimuat ke dalam DataFrame, saya langsung memeriksa distribusi kelasnya untuk memahami struktur data.

# Impor library yang dibutuhkan
import pandas as pd
import mysql.connector
from sklearn.preprocessing import StandardScaler
from sklearn.decomposition import PCA
from imblearn.over_sampling import ADASYN, RandomOverSampler
from imblearn.pipeline import Pipeline
import matplotlib.pyplot as plt
import seaborn as sns
import numpy as np
import warnings

warnings.filterwarnings("ignore", category=FutureWarning)
plt.style.use('seaborn-v0_8-whitegrid')

# LANGKAH 1: KONEKSI KE DATABASE MYSQL
print("--- LANGKAH 1: Menghubungkan ke Database ---")
db_config = {
    'host': 'localhost',
    'user': 'root',
    'password': '',
    'database': 'pertemuan5'
}
table_name = 'ecoli'
try:
    conn = mysql.connector.connect(**db_config)
    query = f"SELECT * FROM {table_name}"
    df = pd.read_sql(query, conn)
    conn.close()
    print(f"✅ Koneksi berhasil! {len(df)} baris data dari tabel '{table_name}' telah dimuat.")

except mysql.connector.Error as e:
    print(f"❌ Error saat koneksi ke MySQL: {e}")
    exit()


# LANGKAH 2: ANALISIS & PERSIAPAN DATA
print("\n--- LANGKAH 2: Menganalisis dan Mempersiapkan Data ---")
feature_columns = ['mcg', 'gvh', 'lip', 'chg', 'aac', 'alm1', 'alm2']
target_column = 'localization_class'

X = df[feature_columns]
y = df[target_column]
print("\n[INFO] Ringkasan struktur data:")
X.info()

print("\n[ANALISIS] Distribusi kelas sebelum penyeimbangan:")
print("Ini menunjukkan adanya ketidakseimbangan (imbalance) yang signifikan.")
class_counts_before = y.value_counts()
print(class_counts_before)

Persiapan Data

Hasil analisis

Hasil ini menunjukkan adanya ketidakseimbangan yang signifikan. Kelas cp memiliki 143 sampel, sementara imL dan imS masing-masing hanya memiliki 2 sampel. Melatih model pada data seperti ini akan menghasilkan performa yang bias dan tidak dapat diandalkan.

Langkah 3: Visualisasi Data dengan PCA#

Untuk mendapatkan gambaran visual dari ketidakseimbangan ini, saya menggunakan PCA (Principal Component Analysis). Teknik ini mereduksi 7 fitur data menjadi 2 komponen utama, sehingga bisa divisualisasikan dalam plot 2D.

Plot tersebut mengonfirmasi bahwa sebaran data didominasi oleh beberapa kelas mayoritas, sedangkan kelas-kelas minoritas sulit untuk diamati.

Langkah 4: Mengatasi Tantangan pada ADASYN#

Saya memilih ADASYN karena kemampuannya untuk beradaptasi dengan distribusi data. Namun, saya menemukan sebuah kendala: ADASYN memerlukan jumlah sampel minimum di setiap kelas untuk dapat berfungsi (secara default, minimal 6 sampel). Karena data saya memiliki kelas dengan hanya 2 dan 5 sampel, algoritma ini tidak bisa langsung dijalankan.

Solusinya adalah pendekatan dua langkah:

  1. Pra-pemrosesan dengan RandomOverSampler: Pertama, saya menggunakan RandomOverSampler untuk menambah jumlah sampel pada kelas-kelas yang sangat sedikit (omL, imL, imS) hingga masing-masing memiliki 6 sampel. Langkah ini hanya bertujuan untuk memenuhi syarat minimal ADASYN.

  2. Implementasi ADASYN: Setelah syarat terpenuhi, saya menerapkan ADASYN pada data yang telah diproses tersebut untuk menyeimbangkan seluruh dataset secara adaptif.

Saya menggunakan Pipeline dari imblearn untuk menjalankan kedua proses ini secara berurutan dan efisien.

# Gabungkan dalam pipeline (urutan penting!)
adasyn = ADASYN(random_state=42)
ros = RandomOverSampler(sampling_strategy=sampling_strategy_step1, random_state=42)
pipeline = Pipeline([('ros', ros), ('adasyn', adasyn)])

# Jalankan proses resampling
X_resampled, y_resampled = pipeline.fit_resample(X, y)

Analisis Hasil: Mengapa Jumlah Sampel ADASYN Bervariasi?

Setelah proses penyeimbangan selesai, saya membandingkan jumlah data per kelas. Di sinilah karakteristik unik ADASYN terlihat.

Output Perbandingan:

Perlu diperhatikan bahwa jumlah sampel setelah proses menjadi bervariasi. Ini bukan kesalahan, melainkan cara kerja ADASYN yang adaptif.

Penjelasannya:

Berbeda dengan SMOTE yang membuat jumlah sampel menjadi sama rata, ADASYN bekerja berdasarkan “tingkat kesulitan” setiap sampel.

  1. ADASYN akan membuat lebih banyak data sintetis di sekitar sampel minoritas yang dikelilingi oleh banyak tetangga dari kelas mayoritas. Sampel-sampel ini dianggap “sulit” untuk dipelajari oleh model.

  2. Sebaliknya, untuk sampel minoritas yang berada di lingkungan yang lebih homogen (dikelilingi oleh kelasnya sendiri), ADASYN akan membuat lebih sedikit data sintetis.

hal Inilah sebabnya mengapa kelas im (menjadi 157) dan om (menjadi 146) mendapat penambahan data paling banyak—kemungkinan karena banyak sampelnya yang berada di perbatasan antar kelas.

Langkah 5: Visualisasi Perbandingan Akhir#

Sebagai bukti akhir dari keberhasilan proses, berikut ini visualisasi perbandingan side-by-side. Plot ini secara spesifik menunjukkan data yang sudah diseimbangkan, yang divisualisasikan menggunakan PCA, dan menandai mana data asli dan mana data hasil buatan (sintetis).

Plot di sebelah kanan menunjukkan distribusi kelas yang jauh lebih seimbang. Titik-titik data sintetis (ditandai dengan X) terlihat mengisi area-area yang sebelumnya kosong, terutama di sekitar perbatasan antar kelas, sesuai dengan cara kerja ADASYN.

Kode Lengkap

Berikut adalah skrip Python lengkap yang saya gunakan dalam proyek ini.

# Impor library yang dibutuhkan
import pandas as pd
import mysql.connector
from sklearn.preprocessing import StandardScaler
from sklearn.decomposition import PCA
from imblearn.over_sampling import ADASYN, RandomOverSampler
from imblearn.pipeline import Pipeline
import matplotlib.pyplot as plt
import seaborn as sns
import numpy as np
import warnings

# Mengabaikan warning yang mungkin muncul dari imblearn
warnings.filterwarnings("ignore", category=FutureWarning)

plt.style.use('seaborn-v0_8-whitegrid')
print("--- LANGKAH 1: Menghubungkan ke Database ---")
db_config = {
    'host': 'localhost',
    'user': 'root',
    'password': '',
    'database': 'pertemuan5'
}
table_name = 'ecoli'

try:
    conn = mysql.connector.connect(**db_config)
    query = f"SELECT * FROM {table_name}"
    df = pd.read_sql(query, conn)
    conn.close()
    print(f"✅ Koneksi berhasil! {len(df)} baris data dari tabel '{table_name}' telah dimuat.")

except mysql.connector.Error as e:
    print(f"❌ Error saat koneksi ke MySQL: {e}")
    exit()

# LANGKAH 2: ANALISIS & PERSIAPAN DATA
print("\n--- LANGKAH 2: Menganalisis dan Mempersiapkan Data ---")
feature_columns = ['mcg', 'gvh', 'lip', 'chg', 'aac', 'alm1', 'alm2']
target_column = 'localization_class'

X = df[feature_columns]
y = df[target_column]

print("\n[INFO] Ringkasan struktur data:")
X.info()

print("\n[ANALISIS] Distribusi kelas sebelum penyeimbangan:")
print("Ini menunjukkan adanya ketidakseimbangan (imbalance) yang signifikan.")
class_counts_before = y.value_counts()
print(class_counts_before)

# LANGKAH 3: TRANSFORMASI PCA PADA DATA ASLI
print("\n--- LANGKAH 3: Melakukan Transformasi PCA pada Data Asli ---")
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

pca = PCA(n_components=2)
X_pca = pca.fit_transform(X_scaled)
pca_df_original = pd.DataFrame(data=X_pca, columns=['Principal Component 1', 'Principal Component 2'])
pca_df_original[target_column] = y
print("✅ PCA pada data asli selesai. Siap untuk visualisasi.")

# LANGKAH 4: PROSES PENYEIMBANGAN DATA DENGAN ADASYN
print("\n--- LANGKAH 4: Menyeimbangkan Data Menggunakan ADASYN (Pendekatan 2 Langkah) ---")
print("Karena ADASYN gagal pada kelas < 6 sampel, kita lakukan 2 langkah:")
print("1. RandomOverSampler untuk menaikkan kelas minoritas menjadi 6 sampel.")
print("2. ADASYN untuk menyeimbangkan seluruh dataset.")
n_original = len(X)

# Langkah 1: untuk RandomOverSampler
# kelas yang jumlahnya di bawah 6
sampling_strategy_step1 = {
    'omL': 6, # dari 5
    'imL': 6, # dari 2
    'imS': 6  # dari 2
}
for cls in class_counts_before.index:
    if cls not in sampling_strategy_step1:
        sampling_strategy_step1[cls] = class_counts_before[cls]

adasyn = ADASYN(random_state=42)
ros = RandomOverSampler(sampling_strategy=sampling_strategy_step1, random_state=42)

# Gabungkan dalam pipeline
pipeline = Pipeline([('ros', ros), ('adasyn', adasyn)])

# Jalankan proses resampling
X_resampled, y_resampled = pipeline.fit_resample(X, y)

print("✅ Proses ADASYN selesai.")

comparison_df = pd.DataFrame({
    'Jumlah Sebelum': class_counts_before,
    'Jumlah Sesudah': y_resampled.value_counts()
}).fillna(0).astype(int)

print("\n[PERBANDINGAN] Jumlah data per kelas sebelum dan sesudah ADASYN:")
print(comparison_df)
print(f"\nTotal data meningkat dari {n_original} menjadi {len(X_resampled)}.")

# LANGKAH 5: VISUALISASI PERBANDINGAN
print("\n--- LANGKAH 5: Menampilkan Visualisasi Perbandingan ---")
print("Plot akan ditampilkan. Mohon tutup jendela plot untuk mengakhiri skrip.")
X_resampled_scaled = scaler.transform(X_resampled)
X_pca_resampled = pca.transform(X_resampled_scaled)

pca_resampled_df = pd.DataFrame(data=X_pca_resampled, columns=['Principal Component 1', 'Principal Component 2'])
pca_resampled_df[target_column] = y_resampled
status = ['Data Asli'] * n_original + ['Data Sintetis (ADASYN)'] * (len(X_resampled) - n_original)
pca_resampled_df['Status'] = status
fig, axes = plt.subplots(1, 2, figsize=(22, 10))
fig.suptitle('Perbandingan Visualisasi Data Ecoli Sebelum dan Sesudah ADASYN', fontsize=20, weight='bold')

# Plot 1: Data Asli (Sebelum)
sns.scatterplot(
    ax=axes[0],
    x='Principal Component 1',
    y='Principal Component 2',
    hue=target_column,
    data=pca_df_original,
    palette='tab10',
    alpha=0.8,
    s=60
)
axes[0].set_title('Sebelum Penyeimbangan Data', fontsize=16)
axes[0].set_xlabel('Principal Component 1', fontsize=12)
axes[0].set_ylabel('Principal Component 2', fontsize=12)
axes[0].legend(title='Kelas')

# Plot 2: Data Seimbang (Sesudah ADASYN)
sns.scatterplot(
    ax=axes[1],
    x='Principal Component 1',
    y='Principal Component 2',
    hue=target_column,
    style='Status',
    data=pca_resampled_df,
    palette='tab10',
    alpha=0.8,
    s=70,
    edgecolor='black',
    linewidth=0.3
)
axes[1].set_title('Setelah Penyeimbangan Data dengan ADASYN', fontsize=16)
axes[1].set_xlabel('Principal Component 1', fontsize=12)
axes[1].set_ylabel('Principal Component 2', fontsize=12)
axes[1].legend(title='Legenda', bbox_to_anchor=(1.02, 1), loc='upper left')

plt.tight_layout(rect=[0, 0.03, 1, 0.95])
plt.show()

print("\n✅ Semua langkah selesai.")
