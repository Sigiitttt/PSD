Proyek UTS PSD#

Dokumentasi ini merinci dua alur kerja (workflow) analisis data yang dilakukan pada dataset E. coli menggunakan KNIME Analytics Platform. Proyek ini mencakup dua analisis utama: deteksi data anomali (outlier) dan penanganan dataset klasifikasi yang tidak seimbang (imbalanced dataset).

  • Dataset: E. coli dari UCI Machine Learning Repository

  • Tools: KNIME Analytics Platform, PostgreSQL

  • Nama: Moch Sigit Aringga

  • NIM: 230411100104

Workflow 1: Deteksi Outlier (Outlier Detection)#

Tujuan untuk mengidentifikasi, menandai, dan memvisualisasikan data anomali (outlier) pada fitur-fitur numerik dari dataset E. coli. Deteksi outlier adalah langkah penting dalam pembersihan data untuk memastikan kualitas dan keandalan model di masa depan.

Alur kerja ini dirancang untuk memproses data dari database, membersihkannya, mendeteksi outlier, dan menampilkan hasilnya secara visual.

Langkah-langkah Analisis#

Pengambilan Data (DB Reader): Data E. coli diambil langsung dari database PostgreSQL.

Pembersihan Data (Missing Value): Langkah pertama adalah menangani nilai yang hilang. Nilai kosong pada kolom numerik diisi dengan nilai rata-rata (Mean) untuk mencegah error pada analisis selanjutnya.

Deteksi Outlier (Numeric Outliers): Fitur-fitur numerik dianalisis menggunakan metode Interquartile Range (IQR). Setiap nilai yang berada di luar rentang Q1 - 1.5IQR dan Q3 + 1.5IQR dianggap sebagai outlier dan nilainya diganti menjadi missing value (?).

Pembuatan Penanda (Rule Engine): Sebuah kolom baru bernama prediction dibuat secara manual. Aturan ditetapkan: jika sebuah baris memiliki nilai missing (karena ditandai sebagai outlier), maka kolom prediction akan bernilai true, dan sebaliknya false.

Pewarnaan (Color Manager): Data diberi warna berdasarkan kolom prediction. Data outlier (true) diberi warna merah, sedangkan data normal (false) diberi warna biru.

Visualisasi (Scatter Plot): Hasilnya divisualisasikan menggunakan Scatter Plot untuk melihat sebaran data. Titik-titik outlier yang berwarna merah akan terlihat jelas terpisah dari data normal.

📊 Hasil dan Interpretasi Visualisasi Scatter Plot berhasil menunjukkan titik-titik data yang merupakan outlier dengan warna yang berbeda. Ini membuktikan bahwa metode IQR efektif dalam mengidentifikasi data yang secara signifikan berbeda dari sisa populasi data.

Workflow 2: Analisis Dataset Tidak Seimbang (Unbalanced Dataset)#

Tujuan

Workflow ini bertujuan untuk menunjukkan masalah yang timbul dari dataset klasifikasi yang tidak seimbang dan membuktikan efektivitas teknik oversampling SMOTE (Synthetic Minority Over-sampling Technique) dalam meningkatkan performa model klasifikasi (Decision Tree).

Masalah Ketidakseimbangan Data Dataset E. coli memiliki 8 kelas localization_class dengan distribusi yang sangat tidak merata. Kelas mayoritas (cp) memiliki jumlah sampel yang jauh lebih banyak daripada kelas-kelas minoritas (imS, imL). Hal ini menyebabkan model cenderung “bias” dan mengabaikan kelas minoritas.

[Sisipkan gambar screenshot output Value Counter yang menunjukkan distribusi kelas di sini]

Diagram Alur Kerja Alur kerja ini dirancang dengan dua cabang paralel untuk membandingkan performa model sebelum dan sesudah penyeimbangan data.

[Sisipkan gambar screenshot workflow unbalanced dataset Anda di sini]

Langkah-langkah Analisis#

Persiapan Data: Sama seperti workflow pertama, data dibersihkan menggunakan Missing Value. Selain itu, node String to Nominal digunakan pada kolom localization_class untuk memastikan tipe datanya dikenali sebagai kolom kelas.

Pembagian Data (Partitioning): Data yang sudah bersih dibagi menjadi dua bagian: 70% untuk data latih (training) dan 30% untuk data uji (testing).

Alur A (Tanpa Balancing): Data latih yang asli (tidak seimbang) digunakan untuk melatih model Decision Tree.

Alur B (Dengan Balancing): Data latih yang sama dimasukkan terlebih dahulu ke node SMOTE dengan opsi Oversample minority classes. Node ini menyeimbangkan data dengan membuat sampel sintetis untuk kelas minoritas hingga jumlahnya sama dengan kelas mayoritas. Data yang sudah seimbang ini kemudian digunakan untuk melatih model Decision Tree kedua.

Pengujian dan Evaluasi (Predictor & Scorer): Kedua model (dari Alur A dan B) diuji menggunakan data uji yang sama persis. Performanya diukur dan dibandingkan menggunakan node Scorer yang menghasilkan Confusion Matrix.

Hasil dan Interpretasi#

Perbandingan Confusion Matrix dari kedua alur menunjukkan perbedaan yang signifikan:

Model Tanpa SMOTE: Menunjukkan performa yang sangat buruk pada kelas-kelas minoritas. Model gagal mengenali hampir semua sampel dari kelas seperti imS, omL, dan pp, meskipun akurasi keseluruhannya mungkin terlihat tinggi.

Model Dengan SMOTE: Menunjukkan peningkatan drastis dalam kemampuan mengenali kelas minoritas. Nilai Recall dan jumlah tebakan benar (diagonal pada matriks) untuk kelas minoritas meningkat secara signifikan.

Ini membuktikan bahwa teknik SMOTE berhasil mengurangi bias model dan membuatnya lebih adil dan efektif dalam menangani dataset yang tidak seimbang.

Kesimpulan#

Proyek ini berhasil mendemonstrasikan dua alur kerja analisis data fundamental menggunakan KNIME. Workflow pertama berhasil mengidentifikasi outlier sebagai bagian dari pembersihan data, sementara workflow kedua membuktikan bahwa penanganan dataset tidak seimbang melalui teknik SMOTE adalah langkah krusial untuk membangun model klasifikasi yang adil dan akurat.
